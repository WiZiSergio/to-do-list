10. tasks.json
Resumen flujo de datos (Data Flow)
El servidor Express lee/escribe las tareas en tasks.json

El frontend hace fetch a /tasks para leerlas

Las operaciones CRUD (crear, editar, borrar) llaman a la API usando fetch

Si la API falla, el frontend usa localStorage como respaldo

Los cambios en tareas actualizan la vista mediante manipulación del DOM

Los módulos están separados para organizar lógica: API, DOM, eventos, almacenamiento
